# Diliq — Requirements

> AI-native company intelligence for venture investors. Type a company name, get an instant, beautifully-rendered investment brief.

## 1. Vision

Diliq compresses what would normally be 30+ minutes of manual research (Crunchbase, news, LinkedIn, competitor sites, SEC, blog posts) into a single live-rendered dashboard generated by Claude. The first user is a partner at a growth-stage VC firm; the product should feel like an unfair advantage in pre-meeting prep, sourcing, and deep dives.

The product's emotional hook is **"this feels like magic."** It should look more like a Linear / Arc / Vercel marketing page than a SaaS dashboard.

## 2. Tech Stack

| Layer | Choice | Rationale |
|---|---|---|
| Framework | **Next.js 15 (App Router)** on Node.js | First-class Vercel target; React Server Components for streaming AI output |
| Language | TypeScript (strict) | Type safety across AI tool schemas |
| Hosting | **Vercel** | Edge functions, streaming, zero-config |
| AI | **Anthropic Claude API** (`@anthropic-ai/sdk`) — `claude-opus-4-7` for deep briefs, `claude-haiku-4-5` for fast tasks (autocomplete, classification) | Best reasoning + tool use; prompt caching for system prompts |
| Web research | Claude's **web search tool** (server tool, no extra vendor) | Keeps stack lean; falls back to Tavily/Exa later if needed |
| Styling | **Tailwind CSS v4** + **shadcn/ui** primitives | Design speed without lock-in |
| Motion | **Framer Motion** + **CSS view transitions** | Page-to-page continuity, glow/shimmer effects |
| Fonts | Geist Sans + Geist Mono (or Inter Display) | Modern AI-product feel |
| Streaming UI | Vercel AI SDK (`ai`, `@ai-sdk/anthropic`) | Streams Claude output into RSC |
| State | URL-as-state + React Query for client cache | Shareable links per company |
| DB (future) | **Supabase** (Postgres + Auth + pgvector) | Defer until we need persistence — see §10 |
| Analytics | Vercel Analytics + PostHog (later) | Usage signal for what features matter |

## 3. Design Principles

The UI is the product's first wow-moment. Treat motion as a feature, not decoration.

- **Dark-first** with a deep near-black canvas (`#0A0A0F`) and a single warm-cool accent gradient (electric violet `#8B5CF6` → cyan `#22D3EE`). Secondary highlight: warm amber `#F59E0B` reserved for fundraising/"new" signal moments — sparingly, so it stays meaningful.
- **Glow, not gloss.** Soft radial gradients behind hero elements; subtle `backdrop-blur` panels; thin 1px borders with `border-white/[0.08]`.
- **Aurora background** on the homepage — animated noise + slow-drifting blurred gradient blobs.
- **Motion hierarchy:**
  - Search submit → input morphs into a "thinking" state with shimmer.
  - Sections **stream in** as Claude produces them (skeleton → typed text → settled).
  - Card hover: lift + ambient glow that follows the cursor.
  - Page transitions use shared-element view transitions for the company logo/name.
- **Type scale:** large display headings (clamp 3rem–5rem), tight tracking, generous line-height for body.
- **Density:** marketing-page airy on the homepage; information-dense on the dashboard but always with whitespace breathing room.
- **Empty states** are first-class — never show a blank panel; always a glowing skeleton or playful placeholder.

## 4. Pages

### 4.1 Homepage `/`
- Animated aurora hero, product name, one-line tagline.
- Massive centered **search bar** (the primary CTA — almost the only thing on screen above the fold).
  - Inline autocomplete suggestions powered by Haiku (low-latency).
  - Press `⌘K` from anywhere to refocus.
  - Recent searches chip row underneath.
- "Trending in VC" or "Recently raised" rail (server-cached, refreshed daily) — gives the page life when empty.
- Footer is minimal.

### 4.2 Company Dashboard `/c/[slug]`
The core artifact. All sections stream in as Claude produces them.

**Hero band:**
- Logo (Clearbit/Brandfetch fallback), legal name, one-line tagline, HQ, employee range, founded year.
- Status pills: stage (Seed → Series F → Public), last round size + date, total raised.
- Primary action buttons: "Save to watchlist," "Share," "Export PDF."

**Sections (each a streamed Claude card):**
1. **What they do** — 2-3 sentence summary + a longer "how it works" expandable.
2. **Founders & key people** — cards with photo, role, prior companies, education, notable signal (e.g., "ex-Stripe payments lead"). Click → opens a person panel with deeper background.
3. **Recent news** — chronological, each item summarized in 1–2 sentences with source link + date. Filter chips: All / Funding / Product / People / Press.
4. **Fundraising history** — visual timeline (rounds, amounts, lead investors, valuation if known). Highlight most recent raise prominently. Pull from Claude web search; cite sources.
5. **Market & competitive landscape** — short market summary, then a grid of competitor cards. **Each competitor card is itself a link to `/c/[competitor-slug]`** so the user can navigate the graph.
6. **Investment signals** *(VC-specific)* — hiring velocity, recent exec hires, customer logos spotted, product launches, regulatory tailwinds/headwinds.
7. **Risks & open questions** — Claude's honest "what would a sharp partner push back on?" — explicitly framed as hypotheses, not facts.
8. **Sources** — footnoted list of every URL Claude used, grouped by section.

**Right rail (sticky on desktop):**
- Quick facts (website, LinkedIn, Crunchbase link).
- "Ask Diliq" — a chat input scoped to this company so she can ask follow-ups ("what's their gross margin profile likely to be?", "draft 5 questions for the CEO").

### 4.3 Watchlist `/watchlist`
- Grid of saved companies with last-updated timestamp.
- Background job re-runs the brief weekly so cards show "🆕 new raise" / "🆕 new exec" badges.
- Pre-Supabase: store in `localStorage`. Post-Supabase: per-user.

### 4.4 Compare `/compare?a=...&b=...`
- Side-by-side of 2–3 companies, same section order.
- Claude generates a "head-to-head" synthesis at the top.

### 4.5 Settings `/settings` *(post-auth)*
- API usage, model preference (speed vs. depth), export defaults.

## 5. AI Architecture

### Pipeline per company brief
1. **Resolver** (Haiku, fast): user query → canonical `{name, domain, slug}`. Disambiguates "Anthropic" vs. "Anthropic Bio."
2. **Planner** (Opus): outputs the section list and assigns research subtasks.
3. **Researcher** (Opus + web search tool): runs each section in parallel as streamed responses. Each section returns structured JSON (validated with Zod) so the UI can render it natively, not as raw markdown.
4. **Synthesizer** (Opus): produces the "Investment signals" and "Risks" sections last, conditioned on everything above.
5. **Citer**: every claim carries a source URL; rendered as small footnote chips.

### Prompt caching
Cache the system prompt + tool definitions (they're large and static) to drop per-request cost and latency. Use the `cache_control` ephemeral marker on the system block.

### Tool use
Claude is given:
- `web_search` (Anthropic-hosted server tool)
- `fetch_url` (server tool, with allowlist + size cap)
- `get_cached_brief(slug)` — internal, returns prior brief if recent
- `save_section(section, json)` — writes structured output incrementally so the UI streams

### Structured output
Every section has a Zod schema. The UI renders from the validated object, not the raw stream. This is what makes it feel like an app, not a chat.

### Guardrails
- Always cite. No citation → don't show the claim.
- Never invent funding numbers; if uncertain, say "reported" or omit.
- Date-stamp every section ("as of 2026-04-24").

## 6. Data Sources (no contracts required at launch)
- Claude web search → covers 80% of needs.
- Public company sites, press releases, LinkedIn public pages, news outlets via fetch.
- Logo: Brandfetch public CDN or Clearbit Logo API (free tier).
- Future paid integrations (gated behind env flags): Crunchbase, PitchBook, Harmonic, SEC EDGAR.

## 7. Environment Variables (Vercel)

Set these in **Vercel Project Settings → Environment Variables** (Production + Preview).

**Required at launch:**
- `ANTHROPIC_API_KEY` — your Claude API key from console.anthropic.com.

**Recommended:**
- `ANTHROPIC_MODEL_DEEP` — defaults to `claude-opus-4-7`.
- `ANTHROPIC_MODEL_FAST` — defaults to `claude-haiku-4-5-20251001`.
- `BRANDFETCH_CLIENT_ID` — for company logos (free tier; optional).
- `NEXT_PUBLIC_APP_URL` — canonical URL: `https://diliq.presler.me`.

**Reserved for later (do not set yet):**
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `RESEND_API_KEY` (transactional email when auth ships)
- `POSTHOG_KEY`

## 9. Non-Functional Requirements
- **First meaningful paint** on dashboard < 1s (skeleton + hero); first section streamed < 4s; full brief < 25s.
- **Mobile-first responsive** — she'll use it on her phone before meetings.
- **Shareable URLs** — every brief is permalinkable; OG image generated per company (`/api/og`).
- **Accessibility** — color contrast AA minimum despite the dark theme; respect `prefers-reduced-motion` (disable aurora + shimmer).
- **Cost guardrail** — per-request token budget; cache briefs for 24h keyed by slug.

## 10. Future: Supabase (defer)
Trigger to add it: when **any** of these is true:
- We need real auth (multi-user, partner team).
- Watchlists must persist across devices.
- We want vector search over prior briefs ("find me companies like Ramp but for healthcare").
- We want background re-runs / scheduled refreshes.

When that day comes, the schema sketch:
- `users` (Supabase Auth)
- `companies (slug, name, domain, last_brief_at, …)`
- `briefs (id, company_slug, json, model, created_at, token_cost)`
- `watchlist_items (user_id, company_slug, notes, added_at)`
- `brief_embeddings` (pgvector) for similarity search
- `events` (audit log of searches, for analytics)

Until then: use Vercel KV (or even just `unstable_cache`) for brief caching, and `localStorage` for watchlists.

## 11. Mission-Critical Features Worth Considering (beyond your list)
Ranked by VC-utility:
1. **"Ask Diliq" chat scoped to a company** — follow-up questions are where investors actually live. ✅ in §4.2.
2. **Investment-signal section** (hiring, exec moves, customer logos) — the leading indicators that don't show up in news headlines. ✅ in §4.2.
3. **Risks / pushback section** — separates a tool from a *thinking partner*. ✅ in §4.2.
4. **Compare mode** — partners constantly weigh deals against each other. ✅ in §4.4.
5. **Weekly digest email** of watchlist changes — the product earns recurring engagement without her having to open it. *(post-Supabase + Resend)*
6. **PDF export / shareable brief link** — for partner meetings & memos.
7. **CRM-light deal notes** per company (private to her). *(post-auth)*
8. **Sourcing mode** — "show me Series B fintechs in LATAM that raised in the last 90 days." A reverse-search using the same AI pipeline. *(v2)*

## 12. Build Phases

**Phase 0 — Scaffold (1 day)**
Next.js + Tailwind + shadcn, Anthropic SDK wired, deployed to Vercel, homepage with static aurora.

**Phase 1 — One brief, end to end (3–5 days)**
Resolver → planner → 3 sections streamed (What / Founders / News). Hardcoded glow/motion. Shippable demo.

**Phase 2 — Full brief (1 week)**
All 8 sections, structured output, citations, sources panel, OG images.

**Phase 3 — Magic polish (3–5 days)**
View transitions, cursor-follow glow, autocomplete, recent searches, mobile pass.

**Phase 4 — VC features (1 week)**
Compare mode, watchlist (localStorage), Ask-Diliq chat, PDF export.

**Phase 5 — Supabase + auth + digest emails** (only when Phase 4 has real usage).

## 13. Decisions

- **Domain:** `diliq.presler.me` (configure as a custom domain in Vercel → Settings → Domains; add a `CNAME` record from `diliq` → `cname.vercel-dns.com` at the `presler.me` registrar).
- **Access:** fully public. No waitlist, no auth gate at launch. Add a soft per-IP rate limit (Vercel KV) so the Anthropic bill stays sane.
- **Data sources:** only free/public sources for now (Claude web search, public web fetch, Brandfetch logos). No PitchBook/Harmonic/Crunchbase API integrations — even if her firm has access, we keep this product self-contained and shareable.
- **Brand colors:** dark canvas `#0A0A0F`, primary gradient violet `#8B5CF6` → cyan `#22D3EE`, signal accent amber `#F59E0B`. Geist Sans + Geist Mono.
